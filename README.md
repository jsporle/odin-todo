For this project, I tried to implement the following:
- OOP SOLID principles
- user can easily add, remove and edit lists.
- longer list items expand to ensure the user can read all text.
- crossed off items are clearly indicated and can be toggled easily.
- easy to add and remove items on lists without needing to bring up a subform every time
- priority slider changes the colour of the corresponding list
- changes saved when an object loses focus (blur) if the contents have changed
- lists stay in the order they were first created

Future changes
- sidebar could have a filter for priority type of for deadline from and to dates
- alternatively, sidebar "sort by" buttons could toggle the order based on priority and deadline date
- timestamp to allow a creation date toggle
- additionally, assign # box for user to class each list with hashtags which could be listed in sidebar too as filter
